NHIỆM VỤ: Sinh data SFT chất lượng cao cho Bigram

Mày là AI agent chuyên sinh data huấn luyện. Nhiệm vụ là sinh
ra 8 file data SFT tiếng Việt chất lượng cao, có reasoning,
sạch, đa dạng. Chạy liên tục từ file 1 đến file 8 không dừng.

═══════════════════════════════════════════════════════════════
QUY TẮC CHUNG - ÁP DỤNG CHO TẤT CẢ FILE
═══════════════════════════════════════════════════════════════

FORMAT MỖI DÒNG:
{"prompt": "...", "response": "..."}

CHẤT LƯỢNG RESPONSE:
- Tiếng Việt có dấu đầy đủ, tự nhiên, không cứng nhắc
- Không viết tắt, không bỏ dấu, không teencode
- Không dùng ký tự — " " …
- Response có chiều sâu: giải thích tại sao, không chỉ nói gì
- Với câu hỏi phức tạp: trình bày từng bước rõ ràng
- Với câu hỏi đơn giản: ngắn gọn súc tích, không lan man
- Không bịa số liệu cụ thể (giá, ngày, tên người thật)
- Nếu không chắc: nói rõ "tôi không chắc chắn về..."

REASONING CAO CẤP:
Với các câu hỏi cần suy luận, response phải thể hiện quá
trình tư duy: phân tích vấn đề, xem xét các góc độ, đưa ra
kết luận có căn cứ. Không nhảy thẳng vào đáp án.

VALIDATE TRƯỚC KHI GHI:
- Mỗi dòng phải là JSON hợp lệ, parse được
- Prompt không rỗng
- Response tối thiểu 20 ký tự
- Không có ký tự — " " … trong bất kỳ field nào
- Loại bỏ dòng không đạt trước khi ghi file

TÊN FILE:
Format: data/<timestamp>-<số thứ tự>.jsonl
Ví dụ: data/20260520-1.jsonl, data/20260520-2.jsonl
Timestamp lấy từ ngày hiện tại định dạng YYYYMMDD.

SỐ DÒNG MỖI FILE: 100 dòng hợp lệ sau khi validate.

FILE 1 — Lập trình và debug (100 dòng)

Phân bổ:
  - Debug lỗi code Python (25): đưa ra đoạn code lỗi, giải
    thích nguyên nhân, sửa và giải thích tại sao sửa vậy
  - Giải thích khái niệm lập trình (25): OOP, đệ quy, độ
    phức tạp thuật toán, design pattern, v.v.
  - Viết hàm/script Python (25): yêu cầu cụ thể, code sạch
    có comment, giải thích logic
  - So sánh kỹ thuật/công nghệ (25): khi nào dùng cái nào,
    trade-off, ví dụ thực tế

Ví dụ mẫu:
  prompt: "Hàm này bị lỗi gì: def tinh_tb(ds): return sum(ds)/len(ds)"
  response: "Hàm có một vấn đề tiềm ẩn: nếu ds là danh sách
  rỗng, len(ds) sẽ bằng 0 và Python sẽ ném ra lỗi
  ZeroDivisionError. Cách sửa: kiểm tra danh sách rỗng trước
  khi tính. [code đã sửa và giải thích]"

FILE 8 — Tổng hợp đa dạng chủ đề (100 dòng)

100 dòng đa dạng, không lặp lại chủ đề đã có trong file 1-7.
Ưu tiên các câu hỏi có reasoning phức tạp, cần nhiều bước
suy luận. Phân bổ tự do miễn đa dạng và chất lượng cao.

═══════════════════════════════════════════════════════════════
QUY TRÌNH THỰC HIỆN
═══════════════════════════════════════════════════════════════

Với mỗi file, thực hiện đúng thứ tự:

  1. Sinh đủ 110 dòng (dư 10% để bù sau khi validate)
  2. Validate từng dòng:
     - JSON parse được
     - Không có ký tự cấm — " " …
     - Response >= 20 ký tự
     - Tiếng Việt có dấu
  3. Loại dòng không đạt, giữ đúng 100 dòng hợp lệ.
     Nếu thiếu thì sinh thêm cho đủ.
  4. Ghi ra file data/<timestamp>-<số>.jsonl
  5. Báo cáo một dòng:
     "File <số>: <tên file> - <số dòng> dòng - loại <n> dòng lỗi"
  6. Tiếp tục file tiếp theo ngay, không dừng chờ xác nhận

Sau khi xong file 8, báo cáo tổng:
  - Danh sách 8 file với đường dẫn đầy đủ
  - Tổng số dòng
  - Tổng số dòng bị loại
